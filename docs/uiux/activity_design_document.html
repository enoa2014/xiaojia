1. 设计目标与原则

核心设计目标

用户体验目标 🎯

  • 快速发现: 用户能在30秒内找到感兴趣的活动
  • 简易报名: 3步内完成活动报名流程
  • 信息清晰: 活动信息一目了然，减少疑问
  • 状态同步: 报名状态实时更新，避免冲突

业务价值目标 📈

  • 参与提升: 活动参与率提升至85%以上
  • 管理效率: 活动管理工作量减少60%
  • 数据洞察: 提供完整的活动数据分析
  • 资源优化: 提高活动资源利用率

设计原则

📱 移动优先

基于微信小程序平台特性，优先考虑移动端体验，确保单手操作友好。

🔒 权限分级

根据用户角色（管理员、社工、志愿者、家长）提供差异化功能和信息展示。

⚡ 性能优先

页面加载时间控制在2秒内，操作响应时间不超过100ms。

核心理念: 让每个家庭都能便捷地参与到关爱大病儿童的活动中来，通过技术手段降低参与门槛，提升活动组织效率。

9. 状态管理设计

活动状态流转

草稿 → 待审核 → 发布 → 报名中 → 即将开始 → 进行中 → 已结束 → 已归档 ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ 编辑中 审核中 已发布 接受报名 开始前准备 活动执行 结果收集 数据归档 ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ 保存 通过 推送 状态更新 签到管理 结束确认 满意度 统计报告 ↓ 调查 拒绝 → 返回草稿

报名状态管理

用户报名状态

  • 未报名: 用户未参与该活动
  • 报名中: 正在填写报名信息
  • 待审核: 报名信息待管理员审核
  • 已确认: 报名成功，等待活动开始
  • 候补中: 加入候补名单
  • 已取消: 用户主动取消报名
  • 已拒绝: 报名被管理员拒绝

状态变更权限

  • 用户自主: 报名、取消(截止前)
  • 系统自动: 超时取消、满员候补
  • 管理员操作: 审核通过/拒绝、手动调整
  • 社工协助: 特殊情况处理

数据同步机制

实时同步策略

// 状态同步处理 const stateSync = { // 报名人数实时更新 participantCount: { updateTrigger: ['registration', 'cancellation'], syncMethod: 'websocket', fallback: 'polling_5s' }, // 活动状态变更 activityStatus: { updateTrigger: ['admin_update', 'time_trigger'], syncMethod: 'push_notification', cache: 'local_storage' }, // 用户报名状态 userRegistrationStatus: { updateTrigger: ['user_action', 'admin_review'], syncMethod: 'immediate_sync', notification: true } };

缓存策略设计

页面缓存

  • 活动列表: 缓存5分钟，后台更新
  • 活动详情: 缓存2分钟，报名状态实时
  • 用户信息: 本地缓存，登录时更新
  • 图片资源: 永久缓存，版本控制

数据更新策略

  • 主动刷新: 用户下拉刷新时
  • 被动更新: 接收到推送通知时
  • 定时更新: 页面焦点时检查更新
  • 增量更新: 仅更新变化的数据
