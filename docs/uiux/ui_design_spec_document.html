1. 设计原则与理念

🎯 核心设计原则

以用户为中心，为大病儿童家庭提供温暖、高效、可信赖的数字化服务体验。

🎯 角色导向设计

根据管理员、社工、志愿者、家长四种角色提供差异化的界面体验，确保每个用户看到最相关的功能和信息。

💚 温暖关怀感

采用温暖的绿色调为主色，传达关爱与希望，营造温馨的服务氛围。

⚡ 高效操作

关键操作路径控制在3步以内，5秒内可达核心功能，减少用户认知负担。

🛡️ 安全可信

清晰的权限状态展示，敏感信息脱敏处理，让用户明确知道数据安全边界。

设计目标

  • 提升工作效率：将档案查询时间从2-3分钟缩短到30秒内
  • 降低学习成本：新用户可在5分钟内掌握核心功能操作
  • 增强使用信心：通过清晰的状态反馈，减少操作错误率至5%以下
  • 强化情感连接：通过温暖的视觉设计，增强用户对公益服务的认同感

按钮变体

9. 交互反馈系统

点击反馈

通用点击反馈

  • 点击时缩放至98%
  • 动画时长150ms
  • 使用ease缓动函数

按钮点击反馈

  • 点击时缩放至95%

加载状态设计

  • 按钮加载: 显示旋转图标，禁用交互
  • 页面加载: 全屏遮罩，居中loading
  • 局部加载: 区域内显示骨架屏
  • 数据加载: 列表项逐个显示动画

操作成功时的对勾绘制动画，增强确认感。

11. 响应式设计

断点系统

断点 设备类型 断点值 适配策略
XS 小屏手机 < 480px 单列布局
SM 大屏手机 480px - 640px 单列或两列
MD 平板竖屏 640px - 768px 两列布局
LG 平板横屏 768px - 1024px 三列布局
XL 桌面设备 > 1024px 多列布局

布局适配原则

  • 移动优先: 从小屏开始设计，向大屏扩展
  • 弹性网格: 使用fr单位和百分比布局
  • 内容优先: 确保内容在任何尺寸下都可读
  • 交互适配: 触摸目标在移动端不小于44px

13. 可访问性设计

对比度要求

WCAG AA 标准

  • 正常文字: 对比度不低于 4.5:1
  • 大文字: 对比度不低于 3:1
  • UI元素: 对比度不低于 3:1

键盘导航支持

/* 焦点指示器 */ .focusable:focus { outline: 3rpx solid var(--color-primary-500); outline-offset: 4rpx; } /* 跳过链接 */ .skip-link { position: absolute; top: -80rpx; left: 24rpx; background: var(--color-primary-600); color: white; }

屏幕阅读器支持

ARIA 标签使用

  • aria-label: 为操作按钮提供描述
  • aria-live: 动态内容更新通知
  • role: 定义元素的语义角色
  • aria-describedby: 提供额外的描述信息

14. 性能优化

CSS 优化策略

减少重绘

  • 使用 transform 替代 position 变化
  • 避免频繁修改引起重排的属性
  • 合理使用 will-change 属性

优化渲染

  • 减少DOM深度和选择器复杂性
  • 使用CSS变量减少重复计算
  • 合并样式变化避免多次重绘

图片优化

响应式图片策略

  • WebP格式: 优先使用，减少50%文件大小
  • 懒加载: 使用Intersection Observer延迟加载
  • 渐进加载: 先显示占位符，再加载实际图片
  • 尺寸优化: 提供多种尺寸适配不同设备

16. 开发指南

命名规范

BEM 命名规范

  • Block: .patient-card
  • Element: .patient-card__title
  • Modifier: .patient-card--featured

状态类命名

  • .is-active - 激活状态
  • .is-loading - 加载状态
  • .is-disabled - 禁用状态
  • .has-error - 错误状态

组件开发规范

组件文件结构

components/ ├── Button/ │ ├── index.js // 导出文件 │ ├── Button.jsx // 主组件 │ ├── Button.module.css // 样式文件 │ └── Button.stories.js // Storybook 文档

代码质量标准

  • CSS选择器深度: 不超过4层
  • 避免 !important: 使用更具体的选择器
  • 使用设计令牌: 避免硬编码数值
  • 通过 Linter: 确保代码规范一致性

18. 总结

💚 设计理念回顾

通过温暖的绿色主调、角色导向的界面设计、高效的操作流程和安全可信的信息展示，为大病儿童家庭提供有温度的数字化服务体验。

关键设计原则

🎯 核心价值

  • 温暖关怀: 绿色主调传达希望与治愈
  • 角色导向: 基于用户角色的差异化设计
  • 高效简洁: 3步内完成核心操作
  • 安全可信: 清晰的权限状态和数据脱敏

📈 预期效果

  • 操作效率提升 50%: 2-3分钟→30秒-1分钟

  1. 阶段性实施: 优先实现核心组件，逐步完善细节

  3. 用户测试验证: 定期收集用户反馈，持续优化体验
  4. 文档维护更新: 随着产品迭代更新设计规范
